Architecture Overview

The HMS utilizes a Microservices architecture to ensure scalability, maintainability, and independent deployment. Each module is a self-contained service communicating via REST APIs. This approach allows for flexible scaling and technology choices for individual services. Security is a primary concern, with authentication and authorization implemented at the API gateway. Data persistence is handled by PostgreSQL, providing reliability and scalability.
